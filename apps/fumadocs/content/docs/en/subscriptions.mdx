## Overview

The plugin supports two subscription flows:

- **Without plan** ã define the billing terms inline per subscription (no card required upfront)
- **With plan** ã create a reusable plan first, then subscribe users to it using a card token

Both produce a `checkoutUrl` to send users to MercadoPago for payment authorization.

---

## Without a plan

Use this when each subscription has different terms (amount, frequency).

```ts
const result = await authClient.mercadoPago.createSubscription({
  reason: "Premium Subscription",
  payerEmail: "user@example.com",
  autoRecurring: {
    frequency: 1,
    frequencyType: "months",    // "days" | "weeks" | "months"
    transactionAmount: 999,
    currencyId: "ARS",
    startDate: "2024-02-01T00:00:00Z",   // optional
    endDate: "2025-02-01T00:00:00Z",     // optional
  },
  backUrl: `${window.location.origin}/subscriptions?status=pending`,  // optional
  idempotencyKey: crypto.randomUUID(),   // optional
})

if (result.data) {
  window.location.href = result.data.checkoutUrl
}
```

## With a plan

### 1. Create the plan (server-side, one time)

Plans are shared configurations that multiple users can subscribe to.

```ts
const plan = await authClient.mercadoPago.createPlan({
  name: "Pro Plan",
  description: "Full access to all pro features",
  autoRecurring: {
    frequency: 1,
    frequencyType: "months",
    transactionAmount: 1999,
    currencyId: "ARS",
  },
})

// plan.data ã { planId, name, url }
```

### 2. Subscribe a user to the plan

This requires a **card token** obtained from the MercadoPago frontend SDK (`@mercadopago/sdk-js`).

```ts
const result = await authClient.mercadoPago.createSubscriptionWithPlan({
  planId: plan.data.planId,
  payerEmail: "user@example.com",
  cardTokenId: "card_token_from_mp_sdk",
  identification: {           // optional, required in some countries
    type: "DNI",
    number: "12345678",
  },
  idempotencyKey: crypto.randomUUID(),
})
```

## Manage subscriptions

### Get a subscription

```ts
const result = await authClient.mercadoPago.getSubscription({
  subscriptionId: "mp_subscription_id",
})
```

### List subscriptions

```ts
const result = await authClient.mercadoPago.getSubscriptions({
  limit: 20,
  offset: 0,
  filters: {
    status: "authorized",   // optional
    planId: "plan_id",      // optional
  },
})
// result.data ã { subscriptions, total, limit, offset }
```

### Pause / resume a subscription

```ts
await authClient.mercadoPago.updateSubscription({
  subscriptionId: "mp_subscription_id",
  status: "paused",       // "paused" | "authorized"
})
```

### Cancel a subscription

```ts
await authClient.mercadoPago.cancelSubscription({
  subscriptionId: "mp_subscription_id",
})
```

## Subscription status values

| Status | Meaning |
|--------|---------|
| `pending` | Created, awaiting user authorization |
| `authorized` | Active and billing |
| `paused` | Temporarily suspended |
| `cancelled` | Permanently cancelled |
| `expired` | Reached end date |
| `unpaid` | Payment failed and not retried |

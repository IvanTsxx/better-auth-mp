## Overview

The plugin registers three additional models via Better Auth's schema system. Your database adapter will create these tables automatically when you run a migration.

---

## `mercadoPagoPayment`

Stores one-time payment records created via `createPayment`.

| Column | Type | Notes |
|--------|------|-------|
| `id` | `string` | Primary key |
| `userId` | `string` | FK → `user.id` |
| `externalReference` | `string` | Unique, used to correlate with webhooks |
| `mercadoPagoPaymentId` | `string?` | Filled after webhook confirms payment |
| `preferenceId` | `string` | MercadoPago preference ID |
| `status` | `string` | Payment status (see [Payment status values](/docs//payments#payment-status-values)) |
| `statusDetail` | `string?` | Detailed status from MercadoPago |
| `amount` | `number` | Total amount charged |
| `currency` | `string` | ISO currency code |
| `paymentMethodId` | `string?` | e.g. `"visa"`, `"account_money"` |
| `paymentTypeId` | `string?` | e.g. `"credit_card"`, `"digital_currency"` |
| `metadata` | `string?` | JSON-serialized extra data |
| `createdAt` | `Date` | Record creation time |
| `updatedAt` | `Date` | Last update time |

---

## `mercadoPagoSubscription`

Stores recurring subscription records.

| Column | Type | Notes |
|--------|------|-------|
| `id` | `string` | Primary key |
| `userId` | `string` | FK → `user.id` |
| `externalReference` | `string` | Unique |
| `mpSubscriptionId` | `string?` | MercadoPago PreApproval ID |
| `planId` | `string?` | MercadoPago plan ID (if created with plan) |
| `status` | `string` | Subscription status |
| `reason` | `string?` | Description shown to user |
| `payerEmail` | `string?` | Subscriber email |
| `transactionAmount` | `number?` | Amount per billing cycle |
| `currencyId` | `string?` | ISO currency code |
| `autoRecurringFrequency` | `number?` | Billing cadence number |
| `autoRecurringFrequencyType` | `string?` | `"days"`, `"weeks"`, or `"months"` |
| `nextPaymentDate` | `Date?` | Next scheduled billing date |
| `metadata` | `string?` | JSON-serialized extra data |
| `createdAt` | `Date` | Record creation time |
| `updatedAt` | `Date` | Last update time |

---

## `mercadoPagoPlan`

Stores reusable subscription plan definitions.

| Column | Type | Notes |
|--------|------|-------|
| `id` | `string` | Primary key |
| `mpPlanId` | `string` | MercadoPago plan ID (unique) |
| `name` | `string` | Plan name |
| `description` | `string?` | Optional plan description |
| `transactionAmount` | `number` | Amount per billing cycle |
| `currencyId` | `string` | ISO currency code |
| `autoRecurringFrequency` | `number` | Billing cadence number |
| `autoRecurringFrequencyType` | `string` | `"days"`, `"weeks"`, or `"months"` |
| `metadata` | `string?` | JSON-serialized extra data |
| `createdAt` | `Date` | Record creation time |
| `updatedAt` | `Date` | Last update time |

---

## Prisma schema example

If you use the Prisma adapter, the plugin auto-generates the models. Your `schema.prisma` will look something like:

```prisma
model MercadoPagoPayment {
  id                    String    @id
  userId                String
  externalReference     String    @unique
  mercadoPagoPaymentId  String?   @unique
  preferenceId          String
  status                String
  statusDetail          String?
  amount                Float
  currency              String
  paymentMethodId       String?
  paymentTypeId         String?
  metadata              String?
  createdAt             DateTime
  updatedAt             DateTime
  user                  User      @relation(fields: [userId], references: [id])
}

model MercadoPagoSubscription {
  id                          String    @id
  userId                      String
  externalReference           String    @unique
  mpSubscriptionId            String?   @unique
  planId                      String?
  status                      String
  reason                      String?
  payerEmail                  String?
  transactionAmount           Float?
  currencyId                  String?
  autoRecurringFrequency      Int?
  autoRecurringFrequencyType  String?
  nextPaymentDate             DateTime?
  metadata                    String?
  createdAt                   DateTime
  updatedAt                   DateTime
  user                        User      @relation(fields: [userId], references: [id])
}

model MercadoPagoPlan {
  id                          String   @id
  mpPlanId                    String   @unique
  name                        String
  description                 String?
  transactionAmount           Float
  currencyId                  String
  autoRecurringFrequency      Int
  autoRecurringFrequencyType  String
  metadata                    String?
  createdAt                   DateTime
  updatedAt                   DateTime
}
```
